AI 代理人 AI Agent 是什麼？
高通整場演講的核心主角
▼
白話解釋

以前的 AI 就像一個等你問問題才說話的客服人員——你問「怎麼訂披薩」，牠告訴你步驟，但不會幫你去做。

AI 代理人（AI Agent）就不一樣了，牠更像一個有手有腳的助理：你說「幫我安排下週的旅遊」，牠會自己查機票、訂飯店、規劃行程，最後把結果呈現給你。

🌟 生活例子

想像你有個超厲害的秘書，你說「幫我準備明天的簡報」，他會自己去找資料、整理重點、排版好，你只需要確認就好。AI 代理人就是這個「數位秘書 Digital Assistant」。

Amon 說，AI 代理人代表了 AI 的「下一個進化」——從被動的工具（Tool）變成主動的工作者（Worker）。這改變了科技公司競爭的方向。

雲端運算 Cloud Computing vs. 邊緣運算 Edge Computing
高通最核心的商業邏輯
▼
白話解釋

雲端運算（Cloud Computing）：你的問題傳到遠方的超大電腦（伺服器 Server），跑完再傳回來。優點是算力強，缺點是要網路、有延遲（Latency）、費用可能很高。

邊緣運算（Edge Computing）：AI 就在你的手機或電腦裡跑，不用傳到雲端。優點是快、不需要網路、更隱私（Privacy），缺點是裝置算力有限。

🌟 生活例子

問 ChatGPT 問題 = 雲端（你的問題飛到美國 OpenAI 的伺服器）。Apple Siri 辨識「嘿 Siri」= 邊緣（直接在你手機裡判斷，不傳出去）。

高通的策略是讓兩者「混合使用（Hybrid）」：簡單任務在裝置跑，複雜任務才送雲端，這樣可以省下 30~60% 的費用（演講中有 Demo 展示）。

Cloud Computing雲端運算 Edge Computing邊緣運算 Hybrid Inference混合式推理（裝置+雲端） Latency延遲（資料傳輸時間差） Privacy資料隱私
為什麼重要？

Amon 說 AI 代理人使用量到 2030 年會爆增 40 倍（40x token growth），如果全靠雲端會貴到不划算，所以邊緣運算的角色會越來越重要，高通剛好在這裡有優勢。

AI 訓練（Training）= 考試前的讀書（需要大量算力，NVIDIA 現在稱霸）。AI 推理（Inference）= 考試當天作答（需要快速低延遲，是 Dragonfly 瞄準的市場）。

Token（代幣 / 文字單元）— AI 的「計費單位」
Amon 說它是「AI 的貨幣 Currency of AI」
▼
白話解釋

Token 是 AI 語言模型（Language Model）處理文字的最小單位，大約 1 個英文單字 = 1~2 個 token，中文 1 個字大概 1~2 個 token。你每次問 AI 問題、AI 每次回答，都在消耗 token。

用 API（應用程式介面）付費呼叫 ChatGPT 就是按 token 計費。Amon 說 AI 代理人（AI Agent）的任務更複雜、對話更長，會讓全球 token 使用量到 2030 年爆增 40 倍。

🌟 生活例子

問 AI「今天幾號？」= 幾個 token，像坐公車投一枚硬幣。叫 AI 代理人「幫我規劃一週旅遊行程並訂好所有票」= 可能要幾千個 token，像搭計程車跑長途。代理人任務多 = token 消耗爆炸性成長。

CPU / GPU / NPU 三者的差異
高通 Snapdragon 晶片裡的三個「大腦」
▼
白話解釋

CPU（中央處理器 Central Processing Unit）：通才，什麼都能做，適合需要邏輯判斷的複雜工作，核心數少但每顆很聰明。你電腦日常運作都靠它。

GPU（圖形處理器 Graphics Processing Unit）：專門做「大量平行運算（Parallel Computing）」，幾千個核心同時算，最初是拿來跑遊戲畫面，現在是 AI 訓練的主力（NVIDIA 就是靠這個稱霸）。

NPU（神經網路處理器 Neural Processing Unit）：專門為 AI 推理（Inference）設計，功耗更低、針對矩陣乘法（Matrix Multiplication）高度優化，適合在手機、筆電等省電場合跑 AI。

🌟 生活例子

CPU = 大學教授（通才、一次處理一個複雜問題）。GPU = 一千個工讀生（同時做很多簡單的重複工作）。NPU = AI 專門老師（只教 AI 課，效率超高但只會這個）。

CPU (Central Processing Unit)中央處理器 GPU (Graphics Processing Unit)圖形處理器 NPU (Neural Processing Unit)神經網路處理器 Parallel Computing平行運算 Matrix Multiplication矩陣乘法（AI 核心運算）
演講連結

高通的 Snapdragon 晶片同時有這三個處理器，Amon 強調 CPU+GPU+NPU 的整合是 AI 代理人需要的「新架構（New Architecture）」。

🔄
生成式 AI Generative AI vs 代理式 AI Agentic AI
整場演講的核心論點
▼
白話解釋

生成式 AI（Generative AI）：你給它一個提示（Prompt），它生成回應（Response）。一問一答，被動（Reactive）。例：「幫我寫一首詩」→ AI 寫完，等你下一個問題。

代理式 AI（Agentic AI）：你給它一個目標（Goal），它自己拆解成子任務（Sub-task）、呼叫工具（Tool Use）、執行步驟、自我修正（Self-correction），一步一步達成目標。主動（Proactive）、持續運作、有「記憶（Memory / Context）」。

🌟 生活例子

生成式 AI = 你問：「怎麼訂機票？」AI 答：「去官網，點選日期...」。代理式 AI = 你說：「幫我訂下週去東京的機票，不超過 15,000 元」→ AI 自己打開瀏覽器、比價、選好、問你確認→ 完成。

Generative AI生成式人工智慧 Agentic AI代理式人工智慧 Prompt提示詞（給 AI 的指令） Tool Use工具呼叫（Agent 操作外部工具） Context / Memory上下文 / 記憶（持續對話記憶） Self-correction自我修正（AI 自動檢查錯誤）
硬體需求差異

代理式 AI 需要持續在背景運行（Background Running）、有長期記憶，所以對硬體的低功耗（Low Power）和持久性要求更高，這正是手機/PC 晶片需要升級的原因。

📡
TOPS 是什麼單位？700 TOPS 有多強？
機器人平台 IQ10 的核心算力規格
▼
白話解釋

TOPS（Tera Operations Per Second）= 每秒執行多少兆次（1兆 = 10¹²）AI 運算。這是衡量 NPU（神經網路處理器）算力的主流單位。數字越大，AI 任務跑越快、可以跑越大的模型。

微軟 Copilot+ PC 認證最低需要 40 TOPS。Dragonwing IQ10 有 700 TOPS，是入門門檻的 17 倍以上。

🌟 生活例子

可以把 TOPS 想成「廚師的切菜速度」。40 TOPS 是一般廚師，能應付家常菜（Copilot+ PC 功能）。700 TOPS 是五星大廚，可以同時服務整個宴會廳。機器人要同時處理視覺（Vision）、動作（Motion）、語音（Speech），需要這樣的算力。

混合式 AI 推理 Hybrid AI Inference 如何省 60% 成本？
Amon 在演講中 Demo 的核心技術
▼
技術解釋

純雲端推理（Cloud-only Inference）的問題：每一次 AI 回應都要把大量資料（對話歷史、上下文 Context、用戶設定）傳到伺服器，不只慢，每次傳輸都消耗大量 token，費用很高。

SoC（System on Chip）片上系統架構
Snapdragon 的底層設計哲學
▼
技術解釋

傳統 PC：CPU、GPU、記憶體控制器（Memory Controller）、網路晶片各自是獨立的晶片，透過匯流排（Bus）連接，傳輸延遲（Latency）高。

SoC（System on Chip，片上系統）：把 CPU、GPU、NPU、記憶體控制器、數據機（Modem）全部整合進單一晶片。好處是延遲極低（晶片內部傳輸速度遠快於外部匯流排）、功耗更低（省去跨晶片傳輸能耗）。

🌟 CS 例子

就像把原本多個 microservice 整合成 monolith 再優化——在合適規模下，內部呼叫（In-process Call）的延遲遠低於網路呼叫（Network Call）。Snapdragon 的 CPU+NPU 共享高頻寬記憶體（LPDDR5x），AI 推理時不需要跨晶片傳資料。

SoC (System on Chip)片上系統 Unified Memory Architecture統一記憶體架構 On-chip Interconnect晶片內部互連 Memory Bandwidth記憶體頻寬 Power Gating電源閘控（動態省電技術） Bus (System Bus)系統匯流排
課堂連結

計算機組織（Computer Organization）、作業系統課程：記憶體階層（Cache Hierarchy）、匯流排架構（Bus Architecture）。SoC 的設計可以看成對傳統「北橋/南橋（Northbridge/Southbridge）」架構的極端整合優化。

LLM 推理的運算瓶頸：Memory Bandwidth（記憶體頻寬）
為什麼大語言模型推理的關鍵不是算力而是記憶體頻寬？
▼
技術解釋

LLM（大型語言模型 Large Language Model）推理是 Memory Bandwidth Bound（記憶體頻寬瓶頸）問題，不是 Compute Bound（算力瓶頸）。每次生成一個 token，模型需要把幾十億的參數（Weights）從 DRAM 載入到計算單元，這個資料搬移速度（Data Movement Speed）才是瓶頸。

Snapdragon X2 Elite 採用 LPDDR5X 記憶體，理論頻寬達 135 GB/s，且 CPU/GPU/NPU 共享這塊記憶體（Unified Memory），不需要像離散 GPU（Discrete GPU）那樣跨 PCIe 傳資料。

🌟 CS 例子

想像 CPU 是廚師（算力超強），記憶體是冰箱（存模型參數 Model Weights），但冰箱到廚房的走廊很窄（記憶體頻寬）。廚師再快，食材搬不過來也沒用。Unified Memory 就是把冰箱直接搬進廚房旁邊。

Memory Bandwidth Bound記憶體頻寬瓶頸 LPDDR5X低功耗雙倍速率記憶體第五代 Unified Memory統一記憶體（CPU/GPU/NPU 共享） Roofline Model屋頂線效能分析模型 Arithmetic Intensity算術強度（運算量/記憶體傳輸量比） Discrete GPU獨立顯示卡

NPU 程式設計模型：ONNX、QNN 與模型量化 Model Quantization
怎麼讓你的 AI 模型在 Snapdragon 上跑
▼
技術解釋

訓練好的 AI 模型（PyTorch / TensorFlow 格式）要跑在 NPU 上，需要幾個步驟：

1. 匯出 ONNX（Open Neural Network Exchange，開放神經網路交換格式）：把模型轉成硬體無關的通用格式。

2. 量化（Quantization）：把模型參數從 float32 壓縮成 int8 或 int4，大幅降低記憶體佔用和計算量（通常效能損失小於 1-3%）。

3. 編譯成 QNN（Qualcomm Neural Network SDK）：使用高通神經網路開發套件把 ONNX 模型編譯成針對 Snapdragon NPU 優化的 QNN 格式，充分利用硬體加速器（Hardware Accelerator）。

🌟 CS 例子

就像寫 Java 程式（PyTorch）→ 編譯成 .class bytecode（ONNX）→ JVM（QNN Runtime）在特定硬體上執行。量化（Quantization）就像從 64-bit double 降到 8-bit int，精度稍降但速度爆增。

Dragonfly 推理晶片架構猜想：脈動陣列 Systolic Array vs 向量處理器 Vector Processor
資料中心 AI 推理晶片的技術路線
▼
技術分析

高通尚未公開 Dragonfly 的技術細節，但根據業界趨勢可以推測兩種主要架構路線：

脈動陣列（Systolic Array）：Google TPU 的方式，矩陣乘法（GEMM, General Matrix Multiply）硬編碼到硬體，Transformer 架構推理極度高效，但靈活性（Flexibility）低。

向量處理器（Vector Processor / VLIW）：更靈活，可以支援不同算子（Operator），犧牲部分效率換取通用性。高通在行動端的 NPU 歷史上傾向後者（Hexagon DSP 架構）。

脈動陣列 = ASIC（Application-Specific IC，只做一件事但做到極致）。向量處理器 = FPGA（可程式化，有更大的適用範圍）。資料中心需要支援各種不同的模型架構，彈性很重要，但推理延遲 SLA 也很嚴格，是個工程取捨（Engineering Trade-off）問題。

熱門方向

🔴 On-device AI / Edge ML Engineer（裝置端 AI / 邊緣機器學習工程師）：讓 AI 模型在手機/嵌入式裝置跑起來。需要：PyTorch → ONNX → TFLite/QNN、模型量化（Quantization）、C++ 優化。高通（Qualcomm）、聯發科（MediaTek）、蘋果（Apple）都在招。

🟡 Robotics AI Engineer（機器人 AI 工程師）：結合 ROS2、電腦視覺（Computer Vision）、強化學習（Reinforcement Learning）設計自主機器人。需要：Python/C++、ROS2、SLAM、RL 基礎。Tesla、Figure AI、小米機器人部門都在搶人。

🟢 AI Infrastructure / MLOps Engineer（AI 基礎設施 / 機器學習維運工程師）：管理 AI 模型的訓練、部署（Deployment）、監控（Monitoring）流程。需要：Kubernetes、Ray、Docker、Prometheus。雲服務商（AWS、Azure、GCP）薪資極高。

🔵 Chip Design / Verification Engineer（晶片設計 / 驗證工程師）：幫高通（Qualcomm）、台積電（TSMC）、聯發科（MediaTek）設計下一代晶片。需要：數位邏輯（Digital Logic）、Verilog/VHDL、VLSI Design Flow。台灣就業市場需求穩定且薪資高。
